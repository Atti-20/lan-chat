## 1. 产品概述

### 1.1 产品定位
一款支持私聊、群聊、文件共享与在线协作的即时通讯平台，兼顾社交沟通与轻量办公需求，提供 Web 端与移动 App 端一致的使用体验。

### 1.2 产品目标
- 打造安全、稳定、易用的多端聊天工具
- 通过阅后即焚、消息撤回等特色功能保障隐私
- 利用文件在线预览减少下载等待，提升效率
- 支持 App 与 Web 双端同时在线，消息实时同步

### 1.3 目标用户
- 需要跨设备沟通的团队与小型组织
- 注重隐私保护的个人用户（如发送敏感信息）
- 有文件共享、轻量审批预览需求的协作场景

## 2. 用户角色
| 角色 | 描述 | 权限 |
|------|------|------|
| 游客 | 未登录用户 | 仅可查看登录/注册页面 |
| 普通用户 | 已注册登录的用户 | 使用所有核心功能：私聊、群聊、文件操作、消息撤回、阅后即焚等 |
| 群主/管理员 | 群组的创建者或被授权者 | 管理群成员、设置群资料、解散群等 |
| 系统管理员 | 后台管理者 | 用户管理、数据监控、内容审核、系统配置 |

## 3. 功能需求

### 3.1 用户注册与登录验证

#### 3.1.1 注册
- 支持手机号/邮箱 + 密码注册
- 必填昵称，唯一 ID 系统自动生成
- 图形验证码或短信/邮件验证码防刷
- 密码强度要求（至少8位，含字母与数字）

#### 3.1.2 登录
- 支持账号密码登录
- 支持手机验证码快捷登录
- 登录失败次数限制，账号临时锁定
- 支持第三方 OAuth2.0 授权登录（如微信、GitHub，可选）

#### 3.1.3 认证与鉴权
- 采用 JWT Token + 刷新令牌机制
- Token 有效期 2 小时，刷新令牌有效期 7 天
- 接口根据角色进行权限校验

#### 3.1.4 多端登录管理
- 同一账号允许 Web 端与 App 端同时登录，互不踢下线
- 同类型终端（如两个手机App）后登录者会提示"已在其他设备登录"，可选择强制下线旧设备
- 用户可在"设备管理"中查看当前登录设备，并主动退出指定设备

### 3.2 好友与私聊

#### 3.2.1 好友关系
- 支持通过 ID/昵称搜索并添加好友
- 好友申请需对方同意，可附验证信息
- 好友列表显示在线状态、最后上线时间
- 支持好友分组、备注名、拉黑/删除

#### 3.2.2 私聊会话
- 一对一实时聊天，消息类型包含：文本、表情、图片、文件、语音（可选）
- 支持消息引用回复
- 会话列表按最后消息时间排序，支持置顶、免打扰
- 未读消息红点提醒，进入会话自动已读

#### 3.2.3 消息记录
- 本地缓存最近消息，支持下拉加载更多历史记录
- 服务端保存消息历史，支持云端漫游（7天或更久可配置）
- 支持会话内关键词搜索

### 3.3 群聊功能

#### 3.3.1 群组管理
- 用户可创建群组，上限200人（可按等级调整）
- 群主可设置群名称、头像、公告、入群验证方式（允许任何人、需审核、禁止加入）
- 群主可转让、设置管理员，管理员具备成员管理权限
- 支持群二维码分享和邀请链接

#### 3.3.2 群聊会话
- 支持与私聊相同的消息类型
- @群成员/所有人 功能，被@用户有特殊提醒
- 群消息免打扰设置
- 显示消息阅读状态（多少人已读，可选功能）

#### 3.3.3 群成员管理
- 群主/管理员可移除成员、禁言指定成员（可设时长）
- 成员可主动退群

### 3.4 文件上传

#### 3.4.1 上传能力
- 支持图片、Word、Excel、PPT、PDF、TXT、压缩包等常见格式
- 单文件大小限制为 100MB（可配置），图片支持批量上传
- 断点续传与秒传功能（根据文件哈希判断）
- 上传进度实时展示

#### 3.4.2 存储与安全
- 文件存储采用对象存储服务（如 MinIO / 阿里云 OSS），数据库仅记录文件路径与元数据
- 上传文件需进行病毒扫描（可选集成）
- 敏感文件可由管理员后台设置访问控制

#### 3.4.3 文件消息
- 聊天中发送文件以卡片形式展示，包含文件名、大小、类型图标
- 支持下载到本地

### 3.5 在线预览

#### 3.5.1 支持格式
- **图片/视频**：缩略图预览，点击全屏查看或播放
- **Office 文档**：docx, xlsx, pptx 通过集成 Office Online Server 或第三方服务（如 OnlyOffice）实现在线预览，无需下载
- **PDF**：直接内嵌浏览器或 WebView 打开
- **文本/代码**：直接在聊天窗口内轻预览

#### 3.5.2 交互体验
- 文件消息提供"预览"按钮，点击后在应用内嵌浏览器或新页面打开
- 预览时支持基本缩放、翻页
- 不能在线预览的格式提供下载指引

### 3.6 消息撤回

#### 3.6.1 撤回规则
- 所有消息（文本、文件、图片等）发送后 **2分钟内** 可撤回
- 撤回后，双方/群内消息位置显示"你撤回了一条消息"或"XX撤回了一条消息"
- 撤回操作仅删除消息展示，已下载到本地的文件不受影响，但链接失效

#### 3.6.2 限制
- 被撤回的消息无法恢复
- 管理员可在后台配置撤回时限

### 3.7 阅后即焚

#### 3.7.1 发送方式
- 私聊和群聊均可发送"阅后即焚"消息，需在发送前手动开启该模式（按钮切换）
- 消息内容支持文本、图片和短视频（文件类禁止焚毁以避免数据丢失）

#### 3.7.2 销毁机制
- 接收方查看消息后，**倒计时5秒**（可配置）自动消失，或离开聊天界面立即消失
- 焚毁后，双方聊天记录中该消息变为"消息已焚毁"占位符，不可再查看
- 服务器在确认所有接收方已读后删除消息内容，仅保留焚毁记录日志

#### 3.7.3 防截屏提醒
- App 端检测到截屏/录屏行为时，系统提示"对方可能已截屏"（需结合系统能力，Web端无法完全防截）
- 对方尝试截图时可选发送提醒，不作为强制功能

### 3.8 多端同步与通知

#### 3.8.1 消息同步
- 同一个账号在不同设备登录时，实时接收消息
- 已在一个设备上读取的消息，其他设备同步显示已读状态
- 阅后即焚消息在多端同步焚毁，一旦一端读后，其他端也不再显示原文

#### 3.8.2 推送通知
- App 端使用厂商推送通道（APNs、Firebase 等）
- Web 端使用 WebSocket 长连接 + 桌面通知 API
- 免打扰时段或会话免打扰下不弹通知

## 4. 非功能性需求

### 4.1 性能
- 消息收发延迟 ≤ 200ms（局域网 / 正常网络）
- 支持单聊群聊消息并发量 ≥ 5000 条/秒
- 文件上传/下载速度影响最小化，利用 CDN 加速

### 4.2 安全
- 传输层使用 HTTPS/WSS 加密
- 敏感信息（密码）使用 BCrypt 加密存储
- 接口防 SQL 注入、XSS、CSRF 攻击
- 文件上传校验 MIME 类型与扩展名白名单，防木马

### 4.3 可用性
- 服务可用性 99.9%，支持水平扩展
- 优雅降级：文件预览服务不可用时自动转为下载

### 4.4 兼容性
- Web 端兼容最新版 Chrome、Firefox、Safari、Edge
- App 端支持 iOS 14+ 与 Android 8.0+

### 4.5 可扩展性
- 微服务架构思想设计模块（消息、用户、文件、群组独立服务）
- 消息系统支持后期接入第三方 AI 助手、机器人

## 5. 数据与接口需求简述

### 5.1 核心数据实体
- 用户表、好友关系表、群组表、群成员表
- 消息表（按会话分表或分库设计，区分私聊/群聊，含消息类型、状态、是否阅后即焚等）
- 文件元数据表、撤回记录表、设备登录表

### 5.2 接口原则
- RESTful API + WebSocket 实时通信
- API 版本管理（/api/v1/...）
- 统一返回格式 `{"code":200, "data":{}, "msg":"success"}`

## 6. 技术栈约束
- **后端核心**：Java 17 + Spring Boot 3.x + MyBatis Plus
- **数据库**：MySQL 8.0 + Redis（缓存/消息队列）
- **消息队列**：RabbitMQ / RocketMQ 用于异步解耦（如文件转换、推送）
- **文件存储**：MinIO / 云OSS
- **实时通信**：WebSocket 结合 STOMP 协议，Netty 网关支持高并发
- **在线预览**：集成 OnlyOffice / Collabora Online 或其他文档预览服务
- **安全框架**：Spring Security + JWT
- **移动端**：React Native / Flutter 或原生开发（可选）
