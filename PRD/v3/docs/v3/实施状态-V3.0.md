# LANChat V3.0 实施状态

> 更新日期：2026-07-19
> 代码基线：`master` / V3.0.0 开发版（正式发布基线仍为 V2.3.0）
> V3.0 总体状态：**macOS 桌面 P0 与工程化代码已落地；安装、签名、公证、真实 Updater/Release 和完整环境回归尚未完成**

## 1. 状态说明

本文只记录可被代码、测试、构建产物、配置或文档证据验证的状态。

| 状态 | 定义 |
|---|---|
| 已实现 | 已存在代码证据并进入当前主链路 |
| 已验证 | 已实现且具有自动测试、构建或可重复验收记录 |
| 进行中 | 已有代码提交，但未满足退出条件 |
| 规划完成 | 范围、方案和验收已明确，尚无完整实现证据 |
| 待开发 | 尚未开始或没有可核验证据 |
| 暂缓 | 不属于当前版本或受外部条件限制 |

V2.3 的能力是 V3.0 的继承基线。V3.0 本轮可把已经进入主链路的 Tauri、原生发现和服务端适配标记为“已实现”，但流水线文件不能替代签名产物、GitHub Actions 运行、真实安装或升级证据。

## 2. 当前结论

### 2.1 已实现代码

- Tauri 2 共用 Vue 桌面构建、运行时桥接和动态节点上下文；
- 关闭隐藏到托盘、托盘打开/重扫/检查更新/退出、单实例、开机自启隐藏、系统通知；
- `node`、`room`、`conversation`、`broadcast` 白名单深链，包含冷启动暂存和第二实例转发；
- 原生 `_lanchat._tcp.local.` 浏览、协议/节点 ID 校验、HTTP 握手、健康探测、延迟与来源状态、最多 32 个缓存节点、手动地址回退；
- 按节点隔离的 Rust HTTP Cookie Jar 负责桌面登录/刷新/退出，Refresh Token 不进入 JavaScript 存储；
- Vue REST、WebSocket、文件资源与 IndexedDB 属主随当前节点切换，Web 同源模式保持兼容；
- 服务端支持 `desktop` 设备类型，`/node/info` 与 mDNS TXT 提供一致的协议和路径能力；
- REST CORS 与 WebSocket 使用明确 Origin 白名单，包含 Tauri 和本地桌面开发 Origin；
- 通用 CI、三平台无签名桌面 smoke build、签名草稿桌面 Release、服务端镜像/Compose 发布包工作流；
- 双实例跨应用消息幂等与浏览器断网文本发件箱恢复 E2E 测试代码。

### 2.2 尚未完成或尚未形成外部证据

- 尚无 `apps/mobile` 的 Android/iOS 可验证交付；
- 尚无二维码节点导入、Server Manager 和桌面设置完整 UI；
- 尚无经 Apple Developer ID 签名、公证并 stapling 的 macOS `.app`/`.dmg`；
- 尚无经真实证书签名与安装验证的 Windows 产物，也没有已验证的 Linux 正式发行包；
- 尚无从旧版本到 V3.0 的真实 Updater 签名、下载、安装与失败回退记录；
- 工作流尚需在 GitHub-hosted runner 上运行并保留成功记录；自托管 mDNS runner 仍需配置；
- 当前 E2E 只覆盖跨实例消息幂等与文本发件箱恢复；Presence、WebRTC、Redis 中断、上传恢复和真实多播尚无完整自动化证据；
- 尚无 Actuator/Prometheus/Grafana/Trace 完整链路；
- 尚无 V3.0 正式 Release。

发布流水线会主动拒绝缺少密钥的任务，不会生成占位公钥或把私钥写入仓库。真实发布至少需要 Tauri Updater 密钥对、Apple Developer ID/公证凭据、Windows 代码签名证书以及对应 GitHub Secrets。

## 5. 里程碑状态

### M1：Desktop Foundation

**状态：核心代码已实现，待 macOS 应用回归与远端 CI**

范围：

- Tauri 工程初始化；
- 共享 Vue 构建接入；
- Native Bridge 最小接口；
- macOS 本地开发与三平台 CI 构建；
- 桌面原生 mDNS；
- 基础 CI。

退出条件：

- `apps/desktop/src-tauri` 存在且可以构建；
- 桌面端可连接 V2.3 服务端并完成登录/聊天；
- 自动列出至少一个局域网节点；
- PR 自动运行基础检查。

### M2：Native Experience

**状态：P0 原生体验代码已实现；P1 Server Manager 与热离线增强待开发**

已实现范围：托盘、通知、单实例、深链、开机自启。未实现范围：Server Manager 初版、热离线增强。

退出条件：桌面客户端可在后台持续运行，通知与导航正确，管理员可查看节点、日志和诊断。

### M3：Release & Verification

**状态：流水线与首批 E2E 代码已实现，正式发布条件未满足**

范围：安装包、代码签名、Tauri Updater、GitHub Release、服务端镜像、双实例和断网 E2E。

退出条件仍未满足：需要 Tag 自动产生真实签名产物、macOS 公证/stapling、Windows 签名、旧版客户端升级，以及完整主链路自动回归通过。

### M4：Mobile

**状态：待开发**

范围：Capacitor Android、局域网权限、通知、文件与任务恢复；iOS Beta 预研。

退出条件：Android 安装包完成登录、聊天、文件、发现/手动连接；iOS 主链路可运行。

### M5：Operations

**状态：待开发**

范围：Actuator、Prometheus、Trace、Grafana、告警、诊断包和备份辅助。

退出条件：关键故障可通过健康、指标、日志和 Trace 复现与定位。

## 7. 证据要求

任何工作流从“待开发/进行中”改为“已实现”至少需要一项代码证据和一项验证证据。

### 7.1 代码证据

| 范围 | 当前证据 |
|---|---|
| 桌面运行时 | `apps/desktop/src-tauri/src/{lib,lifecycle,tray,runtime}.rs` |
| 深链与原生认证 | `apps/desktop/src-tauri/src/{deep_link,native_auth,endpoint}.rs` |
| 原生发现 | `apps/desktop/src-tauri/src/discovery.rs`、`Info.plist`、Tauri capabilities |
| 前端适配 | `frontend/src/platform/`、`useNodeDiscovery.ts`、`useWebSocket.ts`、`api.ts` |
| 服务端协议与 Origin | `LanChatProtocol`、`NodePublicInfo`、`CorsConfig`、`WebSocketConfig` |
| CI/Release | `.github/workflows/`、`scripts/ci/`、`scripts/release/` |
| E2E | `compose.e2e.yaml`、`tests/e2e/` |

### 7.2 验证证据

- 自动测试通过记录；
- 可重复构建命令；
- 安装包、镜像、AAB/IPA 或 Release artifact；
- E2E 日志或报告；
- 截图只能作为补充，不替代代码和测试。

本轮本地基础门禁：

```bash
npm ci --prefix frontend
npm ci --prefix apps/desktop
npm ci --prefix tests/e2e

./mvnw -B test
npm test --prefix frontend
npm run typecheck --prefix frontend
npm run build --prefix frontend

cargo fmt --all --manifest-path apps/desktop/src-tauri/Cargo.toml -- --check
cargo check --locked --manifest-path apps/desktop/src-tauri/Cargo.toml
cargo test --locked --manifest-path apps/desktop/src-tauri/Cargo.toml

npm run typecheck --prefix tests/e2e
docker compose -f compose.yaml -f compose.e2e.yaml config --quiet
git diff --check
```

- Maven 174 个测试和 `lan-chat-server-3.0.0.jar` 打包通过；Vue 6 个单元测试、
  Typecheck、Web Build 与 Desktop Build 通过；

- Apple Silicon 未签名 `LANChat.app` 已实际生成，Bundle 内协议注册、本地网络声明、

## 10. 下一步建议

下一步从“有代码”收敛到“有可发布证据”：

1. 完成本页基础门禁，并保存本轮准确输出；
2. 在两台或两进程 LANChat 节点上用 macOS `.app` 回归发现、手动地址、登录、聊天、文件、房间、广播和节点切换；
3. 实测关闭隐藏、托盘、通知点击、单实例、开机自启和四类深链；
4. 配置 `TAURI_SIGNING_PRIVATE_KEY`、`TAURI_UPDATER_PUBLIC_KEY`、Apple Developer ID/公证 Secrets 与 Windows 证书；
5. 运行 Tag 草稿 Release，验证 macOS 签名、公证、stapling，Windows 签名和 Linux 安装包；
6. 从已安装旧版本执行一次真实 Updater 升级及失败回退；
7. 在 GitHub-hosted runner 跑通 CI/E2E，在自托管多播 runner 补齐真实 mDNS；
8. 补齐 Presence/WebRTC/SYNC/Redis 中断/上传恢复自动化后，再评估 P0 完成态；二维码按 P1 推进；
9. P1/P2 继续保持待开发，不以 P0 交付替代 Mobile、Server Manager 或 Observability。
