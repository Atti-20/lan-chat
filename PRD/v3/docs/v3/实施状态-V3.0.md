# LANChat V3.0 实施状态

> 更新日期：2026-07-17  
> 代码基线：`feature/v2.3` / V2.3.0  
> V3.0 总体状态：**产品化规划与文档基线已完成，功能开发尚未正式进入完成态**

## 1. 状态说明

本文只记录可被代码、测试、构建产物、配置或文档证据验证的状态。

| 状态 | 定义 |
|---|---|
| 已实现 | 已存在代码证据并进入当前主链路 |
| 已验证 | 已实现且具有自动测试、构建或可重复验收记录 |
| 进行中 | 已有代码提交，但未满足退出条件 |
| 规划完成 | 范围、方案和验收已明确，尚无完整实现证据 |
| 待开发 | 尚未开始或没有可核验证据 |
| 暂缓 | 不属于当前版本或受外部条件限制 |

V2.3 的能力可以作为 V3.0 的继承基线标记为“已实现”，但不能因此把 Tauri、Capacitor、自动更新或可观测性标记为完成。

## 2. 当前结论

### 2.1 已完成

- V3.0 研究报告已形成；
- V2.3 现状与 V3.0 增量边界已完成梳理；
- 桌面选型确定为 Tauri 2；
- 移动选型确定为 Capacitor，Android 优先、iOS Beta；
- V3.0 需求分析、功能分析和实施状态文档已建立；
- 目标目录、里程碑、验收标准、风险与发布路线已定义。

### 2.2 尚未完成

- 尚无 `apps/desktop/src-tauri` 的可验证交付；
- 尚无桌面安装包、签名或 Updater 产物；
- 尚无客户端原生 mDNS 浏览器；
- 尚无 `apps/mobile` 的 Android/iOS 可验证交付；
- 尚无 V3.0 GitHub Actions 发布矩阵；
- 尚无 Testcontainers、双实例双客户端自动 E2E 的完整证据；
- 尚无 Actuator/Prometheus/Grafana/Trace 完整链路；
- 尚无 V3.0 正式 Release。

## 5. 里程碑状态

### M1：Desktop Foundation

**状态：规划完成，待开发**

范围：

- Tauri 工程初始化；
- 共享 Vue 构建接入；
- Native Bridge 最小接口；
- Windows 本地开发与构建；
- 桌面原生 mDNS；
- 基础 CI。

退出条件：

- `apps/desktop/src-tauri` 存在且可以构建；
- 桌面端可连接 V2.3 服务端并完成登录/聊天；
- 自动列出至少一个局域网节点；
- PR 自动运行基础检查。

### M2：Native Experience

**状态：待开发**

范围：托盘、通知、单实例、深链、开机自启、Server Manager 初版、热离线增强。

退出条件：桌面客户端可在后台持续运行，通知与导航正确，管理员可查看节点、日志和诊断。

### M3：Release & Verification

**状态：待开发**

范围：安装包、代码签名、Tauri Updater、GitHub Release、服务端镜像、双实例和断网 E2E。

退出条件：Tag 自动产生签名产物；旧版客户端可升级；主链路自动回归通过。

### M4：Mobile

**状态：待开发**

范围：Capacitor Android、局域网权限、通知、文件与任务恢复；iOS Beta 预研。

退出条件：Android 安装包完成登录、聊天、文件、发现/手动连接；iOS 主链路可运行。

### M5：Operations

**状态：待开发**

范围：Actuator、Prometheus、Trace、Grafana、告警、诊断包和备份辅助。

退出条件：关键故障可通过健康、指标、日志和 Trace 复现与定位。

## 7. 证据要求

任何工作流从“待开发/进行中”改为“已实现”至少需要一项代码证据和一项验证证据。

### 7.1 代码证据

- 明确的源码目录、配置或迁移文件；
- 版本化依赖与锁文件；
- 受控权限配置；
- 与现有协议兼容的接口或适配器。

### 7.2 验证证据

- 自动测试通过记录；
- 可重复构建命令；
- 安装包、镜像、AAB/IPA 或 Release artifact；
- E2E 日志或报告；
- 截图只能作为补充，不替代代码和测试。

### 7.3 文档证据

- README 的状态表同步；
- 需求与功能文档的 ID 可映射；
- Release Notes 记录兼容与迁移；
- 已知问题和边界更新。

## 8. Definition of Done

### 8.1 单项功能 DoD

- 需求 ID 与功能设计明确；
- 代码评审通过；
- 单元/集成/端到端测试按风险覆盖；
- 权限与安全边界验证；
- 错误和降级路径可用；
- 文档与状态更新；
- 不引入未说明的数据迁移或兼容破坏。

### 8.2 V3.0 Release Candidate DoD

- Windows 桌面和 Android 至少有可安装构建；
- 桌面原生发现及回退路径可用；
- V2.3 主链路在 Desktop/Mobile 中通过回归；
- 自动更新与签名验证通过；
- 双实例与断网恢复 E2E 通过；
- 服务端镜像和 Compose 发布包可重复生成；
- Health 与核心指标可采集；
- README、需求、功能、状态和 Release Notes 一致。

## 9. 当前风险

| 风险 | 当前判断 | 处理状态 |
|---|---|---|
| Tauri 与现有前端路径/路由适配 | 中 | 需要在 M1 先做最小 PoC |
| Windows WebView2 环境差异 | 中 | 需要安装与兼容测试矩阵 |
| mDNS 在 Docker/企业网络中不可达 | 高 | 必须保留多种回退路径 |
| Android/iOS 局域网权限 | 高 | Android 优先，iOS Beta |
| 代码签名证书与成本 | 中高 | 在 M3 前确定证书与受保护环境 |
| 多平台并行导致范围膨胀 | 高 | 严格按 Windows→Android→iOS 顺序 |
| 深离线凭证安全 | 高 | V3.0 不承诺完全冷启动离线登录 |
| 云端 runner 无多播环境 | 中 | 自托管 runner 或局域网实验机 |

## 10. 下一步建议

建议从一个可验收的最小纵向切片开始：

1. 创建 `feature/v3.0-desktop-foundation` 分支；
2. 建立 `apps/desktop/src-tauri`；
3. 让 Tauri 直接消费现有 `frontend/dist`；
4. 提供 Native Bridge 的 `runtime`、`discoverNodes`、`notify` 最小接口；
5. 完成 Windows 开发启动与无签名安装包；
6. 添加 PR 基础 CI；
7. 用两个可访问节点验证发现、登录、聊天和重连；
8. 更新本文状态为“进行中”并附代码/测试路径。
