## 1. 文档概述

### 1.1 目的
本文档对网络聊天室应用的核心功能进行详细分析，细化业务规则、交互流程、状态变更及异常处理，作为开发、测试及UI设计的直接输入。

### 1.2 范围
涵盖用户注册登录、多端管理、好友私聊、群聊、文件上传、在线预览、消息撤回、阅后即焚、通知同步等全部功能模块。

## 3. 详细功能分析

### 3.1 用户账户与认证

#### 3.1.1 注册
**功能描述**：用户通过手机号或邮箱创建新账号，设置密码和昵称。

**参与者**：游客

**前置条件**：无

**基本流程**：
1. 用户输入手机号/邮箱，获取验证码。
2. 系统校验账号唯一性，发送6位数字验证码（有效期5分钟，60秒内不可重复发送）。
3. 用户填写昵称、密码（需符合强度：8-20位，含字母和数字）、确认密码。
4. 用户同意用户协议后提交。
5. 系统创建账号，自动分配唯一数字ID，返回注册成功，引导至登录页。

**业务规则**：
- 同一IP/设备1小时内最多申请3次验证码，超出需图形滑块验证。
- 昵称长度2-16字符，支持中英文及常用符号，不可包含敏感词。
- 注册即视为同意协议，需在界面勾选。
- 注册后默认生成空白好友列表和系统消息会话。

**异常处理**：
- 验证码错误：提示并清空输入框，保留发送倒计时。
- 网络超时：提示"网络连接失败，请稍后重试"，保留已填写信息。

#### 3.1.2 登录
**功能描述**：用户凭账号密码或手机验证码登录系统。

**参与者**：已注册用户

**基本流程（密码登录）**：
1. 用户输入手机号/邮箱/账号ID + 密码。
2. 系统校验密码，连续5次错误则锁定账号30分钟。
3. 验证通过后，服务端签发Access Token（2小时）和Refresh Token（7天），返回用户基本信息。
4. 客户端存储Token，后续请求携带Authorization头。

**多端登录规则**：
- 同一账号允许在不同设备类型下同时登录：最多1个Web端 + 1个移动App端。
- 若同一类型设备（如两部手机）登录，后登录设备会收到提示："您的账号已在其他手机设备登录，是否继续登录？"，用户可选择"继续登录"（原设备被强制下线）或"取消"。
- 被踢下线的设备收到系统消息："您的账号在其他设备登录，您已被迫下线"，并清空Token。

**异常处理**：
- 账号锁定：提示"账号已被锁定，请30分钟后重试或找回密码"。
- Refresh Token过期：提示重新登录。

#### 3.1.3 多端设备管理
**功能描述**：用户可查看当前登录设备列表，并主动下线指定设备。

**入口**：设置 -> 设备管理。

**列表展示信息**：
- 设备类型（Windows Chrome / iPhone App / Android App 等）
- 登录时间
- IP归属地（如广东东莞）
- 当前设备标识（本机）

**操作**：
- 点击"下线"，该设备Token立即失效，对应客户端收到强制下线通知并跳转登录页。
- 所有Web端在同一浏览器内用不同标签页登录视为同一设备。

### 3.2 好友与私聊

#### 3.2.1 添加好友
**功能描述**：用户通过精确搜索用户ID或昵称发起好友申请。

**流程**：
1. 用户A在添加好友界面输入对方ID/昵称搜索。
2. 系统返回匹配的用户列表（仅显示ID、昵称、头像），隐藏敏感信息。
3. 用户A点击"添加"，可附带验证消息（20字内）。
4. 用户B在"新朋友"列表收到好友申请，可选择"同意"或"拒绝"。
5. 若同意，双方成为好友，各自好友列表出现对方，并可开始私聊。若拒绝，A不会收到通知，但可再次申请。

**业务规则**：
- 已为好友则搜索结果显示"已添加"。
- 黑名单内用户不可互相申请，需先移除黑名单。
- 申请有效期为7天，过期自动失效。

#### 3.2.2 好友管理
- **好友列表**：按最后通信时间倒序，支持置顶（最多5个）、备注名（修改后仅自己可见）、免打扰开关。
- **删除好友**：双向解除关系，聊天记录保留但不可再发消息（除非重新添加）。
- **拉黑**：拉黑后对方无法向你发消息，无法查看你的资料更新；可在黑名单管理页移出。

#### 3.2.3 私聊会话
**消息类型**：文本、表情（内置emoji）、图片、文件、语音（可选扩展）。

**会话行为**：
- 打开会话即发送已读回执，未读消息红点消失。
- 支持消息引用：长按消息选择"引用"，回复时显示引用原文摘要。
- 发送消息时若接收方离线，消息存入离线队列，上线后拉取。

**消息状态**（发送方视角）：
- 发送中（时钟图标）
- 已发送（单勾）
- 对方已读（双蓝勾，仅当对方打开会话时标记）

### 3.3 群聊管理

#### 3.3.1 创建群组
**流程**：
1. 用户点击"创建群聊"，设置群名称（必填，2-20字符）、群头像（可选）、群简介。
2. 选择入群验证方式：允许任何人加入 / 需管理员审核 / 不允许任何人加入（仅通过邀请链接）。
3. 创建成功后，系统生成唯一群号及分享二维码/链接，用户可邀请好友。

**权限说明**：
- 创建者默认为群主，不可退群（需转让群主后退出）。
- 群主拥有所有管理权限，可任命最多3名管理员。

#### 3.3.2 成员管理
**添加成员**：
- 群主/管理员通过ID直接添加好友进群（无需同意）。
- 分享链接/二维码，对方扫码后按入群方式决定直接进入或需审核。

**移除成员**：
- 群主/管理员可将普通成员移出，被移出者收到系统提示"你已被移出群聊"。
- 不可移除群主。

**禁言**：
- 管理员可对单个成员设置禁言（1分钟~30天），禁言期间该成员无法发送任何群消息，尝试发送时提示"你已被禁言，剩余X小时"。

**退出与解散**：
- 普通成员可主动退群。
- 群主退群前必须转让群主给任意管理员，若无管理员则需先任命；转让后自动变为普通成员，可退群。
- 群主选择"解散群聊"则直接解散，所有成员被移出，群聊数据保留但不可访问。

#### 3.3.3 群聊特殊消息
- **@功能**：在输入框输入@唤起成员列表，发送后该成员收到强提醒（消息高亮，推送通知显示"有人@你"）。
- **@所有人**：仅群主和管理员可使用，全员强提醒。
- **群公告**：置顶显示，编辑时支持文本，发布后所有成员可见，并收到系统消息提醒。

### 3.4 消息增强功能

#### 3.4.1 消息撤回
**规则**：
- 所有类型消息发送后 **2分钟内** 可撤回。
- 撤回操作仅消息发送者可执行。
- 撤回后，原消息位置显示灰底小字"你撤回了一条消息"（自己视角）或"XXX撤回了一条消息"（对方视角）。
- 已撤回的消息不再支持任何操作（引用、转发等）。
- 被撤回的文件/图片：服务器删除对应文件存储（若有其他消息引用该文件则不删除），已下载到本地的文件不受影响，但通过消息链接无法再次下载。

**特殊场景**：
- 阅后即焚消息未被阅读前可撤回，撤回后双方均无法查看；已被阅读则无法撤回。
- 群聊中管理员不可撤回他人消息。

#### 3.4.2 阅后即焚
**发送流程**：
1. 用户在聊天输入框点击"焚毁模式"图标（火焰），图标高亮，进入模式。
2. 在模式下发送的文本、图片、短视频（不超过15秒）均标记为焚毁消息。
3. 发送后该消息在发送方显示倒计时圆环（5秒），接收方消息卡片带火焰标识。
4. 接收方进入会话界面并看到该消息时，开始5秒可见倒计时（若直接滚动到该消息也触发），倒计时结束后内容替换为"消息已焚毁"占位。
5. 接收方在倒计时期间离开会话或切到后台，消息立即焚毁，再次进入时显示占位。
6. 所有目标用户均已读且焚毁后，服务端删除消息内容，保留焚毁记录。

**多端同步**：
- 一端触发焚毁，其他端同步更新为占位符。未读的焚毁消息在其他端不显示倒计时，但一旦进入会话即变为占位。

**截屏提醒（App端）**：
- 当界面展示焚毁消息且倒计时进行中，系统检测到截屏行为，消息详情记录"对方可能已截屏"，并推送给发送方一条系统消息："对方可能已截屏"。不阻止截屏。
- Web端不做截屏检测。

**限制**：
- 文件消息、语音消息不可设为焚毁。
- 群聊中焚毁消息需所有在线成员均已读并焚毁后服务器才删除，离线成员未读不阻塞删除，待其上线该消息直接显示为"消息已焚毁"。

### 3.5 文件服务

#### 3.5.1 文件上传
**流程**：
1. 用户在聊天界面点击文件图标，选择本地文件（支持相册/相机/文件管理器）。
2. 前端计算文件MD5，调用"检查文件是否存在"接口。
3. 若文件已存在于服务端（秒传），直接返回文件访问地址和元数据，构建文件消息。
4. 若不存在，开始上传。单文件≤100MB直接上传；＞100MB提示大小超限。
5. 大文件（>2MB）自动分片上传，并行分片，显示进度条。
6. 上传完毕，服务端进行格式校验、病毒扫描（若集成），保存到对象存储，返回文件ID、名称、大小、类型图标。
7. 前端生成文件卡片消息发送。

**规则**：
- 支持格式：jpg/png/gif/mp4/doc/docx/xls/xlsx/ppt/pptx/pdf/txt/zip/rar。
- 图片自动生成缩略图（300x300），用于卡片展示。
- 文件存储在隔离的Bucket中，文件名使用UUID重命名。

#### 3.5.2 文件消息卡片
- 展示：文件类型图标 + 文件名 + 文件大小。
- 操作按钮：预览（支持预览格式时显示）、下载、另存为。
- 点击卡片默认行为：若可预览则打开预览，否则触发下载。

#### 3.5.3 在线预览
**支持格式与实现**：
- 图片/视频：点击放大查看，支持左右滑动，视频内嵌播放器播放。
- Office文档（docx/xlsx/pptx）：集成OnlyOffice或Collabora Online进行文档渲染，在应用内通过WebView或新标签页加载预览。
- PDF：内嵌PDF.js查看器，支持页码跳转。
- 文本/代码：直接弹出浮层显示内容，支持高亮。

**安全与鉴权**：
- 预览链接为临时签名URL，有效期10分钟，与用户Token绑定。
- 不允许直接通过外部链接访问源文件。
- 预览时禁用下载（通过渲染服务控制，无法完全杜绝，仅增加门槛）。

**异常降级**：
- 预览服务不可用或超时，弹窗提示"预览加载失败，是否下载文件？"。

### 3.6 多端同步与通知

#### 3.6.1 消息同步
- 所有在线设备通过WebSocket接收消息，离线设备上线后通过API拉取遗漏消息。
- 已读状态同步：一端打开会话后，服务端广播已读状态给其他端，同步消除红点。
- 撤回、焚毁等变更实时推送，各端UI即时刷新。

#### 3.6.2 推送通知
**场景**：
- App在后台/锁屏时，新消息通过厂商推送通道（APNs/FCM）送达，点击通知进入对应会话。
- Web端关闭或切到后台标签页时，使用Service Worker触发桌面通知（需用户授权）。

**免打扰逻辑**：
- 全局免打扰时段设置（如22:00-8:00），期间不弹推送，仅留存未读计数。
- 单会话免打扰开启后，新消息不弹通知，但会话列表保留红点。

## 4. 非功能性补充（分析层面）
- **并发控制**：消息发送采用乐观锁与消息队列，保证先发先达。文件秒传哈希索引需高性能Redis支持。
- **数据一致性**：消息已读状态最终一致性，短时间内可能看到双勾不对称，但刷新后一致。
- **安全策略**：所有操作均需验证Token归属（如撤回只能本人）。文件上传接口限制并发连接数防止滥用。
- **埋点需求**：消息发送成功率、阅后即焚使用率、预览成功率等需埋点，为迭代提供依据。

## 5. 附录：核心异常场景汇总

| 场景 | 处理方式 |
|------|----------|
| 发送消息时接收方已被删除好友 | 返回错误"双方非好友关系"，消息无法发送 |
| 群聊中发送消息，自己已被移出群 | 消息发送失败，并提示"你已不在该群" |
| 文件上传中断网 | 支持断点续传，保留已上传分片，续传时从断点继续 |
| 撤回超时 | 撤回按钮置灰，点击提示"超过2分钟无法撤回" |
| 焚毁消息倒计时内App崩溃 | 重启后，该消息根据服务端状态显示"消息已焚毁"或未读直接焚毁 |
| 在线预览加载Office时转换失败 | 提示"文档格式暂不支持预览，建议下载查看" |
| 多端被踢下线时正在编辑消息 | 本地草稿保存，重新登录后可恢复草稿内容 |
