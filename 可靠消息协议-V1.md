# LanChat WebSocket 可靠消息协议 V1

## 1. 连接与认证

客户端连接：

```text
ws://<node>/ws/chat
```

连接成功后 10 秒内发送 `AUTH`。令牌不得放入查询参数。

```json
{
  "version": 1,
  "event": "AUTH",
  "requestId": "req_01",
  "timestamp": 1784077200000,
  "payload": {
    "token": "<access-token>"
  }
}
```

认证成功返回 `AUTH_OK`；令牌过期返回 `TOKEN_EXPIRED`；设备会话被吊销返回 `FORCE_LOGOUT`。

## 2. 统一信封

```json
{
  "version": 1,
  "event": "CHAT_SEND",
  "requestId": "req_02",
  "clientMsgId": "client_01",
  "conversationId": "private:7:9",
  "timestamp": 1784077200000,
  "payload": {}
}
```

| 字段 | 规则 |
|---|---|
| `version` | 当前固定为 `1` |
| `event` | 大写事件名 |
| `requestId` | 请求型事件必填，用于关联响应和日志 |
| `clientMsgId` | `CHAT_SEND` 必填；同一发送者范围内唯一 |
| `conversationId` | 私聊或群聊事件必填 |
| `timestamp` | 客户端毫秒时间，仅用于诊断，不参与消息顺序 |
| `payload` | 事件数据对象 |

## 3. 发送与确认

私聊发送：

```json
{
  "version": 1,
  "event": "CHAT_SEND",
  "requestId": "req_02",
  "clientMsgId": "client_01",
  "conversationId": "private:7:9",
  "timestamp": 1784077200000,
  "payload": {
    "toUserId": 9,
    "contentType": "text",
    "content": "hello",
    "isBurn": false
  }
}
```

群聊把 `toUserId` 替换为 `groupId`，会话 ID 使用 `group:<id>`。

事务提交成功后返回：

```json
{
  "version": 1,
  "event": "CHAT_ACK",
  "requestId": "req_02",
  "clientMsgId": "client_01",
  "conversationId": "private:7:9",
  "timestamp": 1784077200108,
  "payload": {
    "clientMsgId": "client_01",
    "messageId": "server_01",
    "conversationId": "private:7:9",
    "sequence": 128,
    "serverTime": 1784077200108,
    "duplicated": false
  }
}
```

ACK 超时可以重发，但必须复用相同 `clientMsgId`。服务端若已保存，会返回原 `messageId` 和 `sequence`，同时令 `duplicated=true`。

接收端通过 `CHAT_DELIVER` 获得完整消息。服务端时间和 `sequence` 是最终排序依据。

## 4. 重连同步

认证成功后，客户端为每个会话提交本地最后接收的序列：

```json
{
  "version": 1,
  "event": "SYNC_REQUEST",
  "requestId": "req_sync_01",
  "timestamp": 1784077300000,
  "payload": {
    "positions": {
      "private:7:9": 127,
      "group:3": 42
    },
    "limit": 100
  }
}
```

响应包含缺失消息、服务端最新位置、拒绝访问的会话和分页标记：

```json
{
  "version": 1,
  "event": "SYNC_RESPONSE",
  "requestId": "req_sync_01",
  "timestamp": 1784077300100,
  "payload": {
    "messages": [],
    "latestPositions": {
      "private:7:9": 130,
      "group:3": 42
    },
    "deniedConversationIds": [],
    "hasMore": false
  }
}
```

当 `hasMore=true` 时，客户端保存本批最大序列并使用新位置继续请求。

## 5. 其他事件

| 事件 | 方向 | 用途 |
|---|---|---|
| `PING` / `PONG` | 双向 | 心跳和 RTT |
| `CHAT_READ` | 双向 | 推进会话最后已读序列 |
| `CHAT_RECALL` | 双向 | 两分钟内撤回自己的消息 |
| `CHAT_BURN` | 双向 | 焚毁有权访问的阅后即焚消息 |
| `TYPING_START` / `TYPING_STOP` | 双向 | 短期输入状态 |
| `ONLINE_LIST` | 服务端到客户端 | 已脱敏在线成员列表 |
| `PRESENCE_CHANGED` | 服务端到客户端 | 上下线状态变化 |
| `FRIEND_CHANGED` | 服务端到客户端 | 好友关系变化 |
| `ERROR` | 服务端到客户端 | 结构化业务或协议错误 |

## 6. 错误处理

`ERROR.payload` 包含：

```json
{
  "code": "BUSINESS_REJECTED",
  "message": "无权在该会话发送消息",
  "retryable": false
}
```

若错误携带 `clientMsgId`，客户端应更新对应本地消息，而不是新增系统消息。`retryable=false` 的权限或参数错误应保留为失败状态，等待用户修改或取消。

## 7. 当前限制

- 当前服务端为单实例内存 WebSocket 路由；
- `CHAT_DELIVER` 已提供，精细送达回执尚未单独落表；
- 单次同步最多处理 100 个会话、全局最多返回 200 条（每会话上限同为 200 条）；
- 文件先经 REST 上传成功，再通过可靠消息引用；离线文件队列尚未实现。
