## 1. 文档概述

### 1.1 编写目的

本文档在需求分析的基础上，将 LanChat 的产品能力细化为可实现、可测试的功能模块，明确每项功能的参与角色、入口、前置条件、操作流程、状态变化、业务规则、异常处理和验收标准。

本文档主要解决以下问题：

- 用户如何完成具体操作；
- 前端需要展示哪些页面、状态和反馈；
- 后端需要校验哪些业务规则；
- WebSocket 和网络层需要处理哪些事件；
- 数据库需要保存哪些关键状态；
- 测试人员如何判断功能是否达到要求。

### 1.2 产品功能主线

LanChat 不再以开放社交型即时通信为主要方向，而是围绕以下主线展开：

1. 用户连接组织自建的 LanChat 节点；
2. 在局域网内完成私聊、群聊、广播和文件交换；
3. 网络中断后继续使用本地缓存和离线发件箱；
4. 连接恢复后通过消息序列号补拉遗漏内容；
5. 通过消息唯一标识、ACK、幂等校验和状态机保证消息可靠；
6. 组织管理员控制用户、设备、数据保留、文件权限和同步范围；
7. 项目现场或应急场景通过临时房间和确认型广播快速协作；
8. 管理员和用户可以查看当前网络、传输路径和同步状态。

### 1.3 功能状态说明

本文档描述目标功能，不代表代码仓库中全部功能已经完成。项目必须在 README 中使用下列状态标识：

| 状态 | 含义 |
|---|---|
| 已实现 | 已完成开发，并通过基本测试 |
| 部分实现 | 已存在代码，但流程、异常或测试不完整 |
| 开发中 | 正在实现，接口或页面可能变化 |
| 规划中 | 已完成需求和设计，尚未编码 |
| 暂缓 | 当前版本不实现 |

## 3. 全局功能规则

### 3.1 统一资源范围

所有业务资源必须归属于组织：

- 用户属于组织；
- 会话属于组织；
- 群组和临时房间属于组织；
- 文件对象具有组织归属；
- 广播属于组织；
- 节点可以承载一个或多个组织，但 MVP 建议一个节点对应一个组织。

任何接口都必须校验当前用户是否属于资源所在组织。

### 3.2 统一会话模型

私聊、群聊和临时房间统一抽象为会话：

| 会话类型 | 说明 |
|---|---|
| PRIVATE | 两名组织成员之间的私聊 |
| GROUP | 长期存在的群组会话 |
| TEMPORARY | 有明确到期时间和销毁策略的临时房间 |
| SYSTEM | 系统通知、设备下线和安全提醒 |
| BROADCAST | 确认型广播的展示会话或聚合入口 |

统一会话模型应支持：

- 成员列表；
- 成员角色；
- 最后一条消息；
- 会话序列号；
- 成员最后已读序列号；
- 置顶；
- 免打扰；
- 未读数量；
- 加入和退出时间；
- 会话状态。

### 3.3 统一错误反馈

前端不得只显示“系统异常”。错误反馈应至少区分：

- 网络不可用；
- 节点不可达；
- 登录状态失效；
- 权限不足；
- 已不在会话中；
- 对方不可接收消息；
- 消息重复；
- 文件类型不支持；
- 文件超过限制；
- 存储空间不足；
- 上传任务过期；
- 同步冲突；
- 服务降级；
- 未知系统错误。

### 3.4 全局操作反馈

- 创建、修改、删除操作必须显示成功或失败结果；
- 长任务必须显示进度；
- 可重试任务必须提供重试入口；
- 高风险操作必须二次确认；
- 离线状态下不可执行的操作应直接说明原因；
- 权限变化后，前端应及时隐藏或禁用对应操作；
- 禁用按钮必须提供原因说明，不能只显示灰色。

## 4. 组织与节点管理

## 4.1 组织初始化

### 4.1.1 功能描述

首次部署 LanChat 节点时，由节点管理员创建组织并完成基础安全配置。

### 4.1.2 参与角色

- 节点管理员；
- 组织所有者。

### 4.1.3 前置条件

- 服务端已经启动；
- 数据库连接正常；
- 当前节点尚未完成初始化；
- 初始化入口仅允许从可信网络或本机访问。

### 4.1.4 页面内容

初始化向导包括：

1. 节点信息；
2. 组织信息；
3. 管理员账号；
4. 运行模式；
5. 数据保留策略；
6. 文件存储配置；
7. 配置确认。

### 4.1.5 基本流程

1. 系统检测数据库中不存在已初始化组织；
2. 跳转组织初始化页；
3. 管理员填写组织名称、组织标识、时区和语言；
4. 设置首个组织所有者账号和密码；
5. 选择本地独立、局域网优先或混合同步模式；
6. 配置普通消息、文件和审计日志保存期限；
7. 测试数据库、Redis 和文件存储状态；
8. 用户确认配置；
9. 服务端在事务中创建组织、所有者和默认角色；
10. 标记节点为已初始化；
11. 关闭公开初始化入口；
12. 跳转登录页。

### 4.1.6 业务规则

- 组织标识在当前节点内唯一；
- 首个管理员密码必须符合强度要求；
- 不得使用默认 JWT 密钥；
- 本地独立模式下不得默认启用外部同步服务；
- 初始化完成后普通用户不得重新进入初始化页面；
- 重新初始化必须由节点管理员执行清空或恢复操作。

### 4.1.7 异常处理

| 异常 | 处理 |
|---|---|
| 数据库不可用 | 阻止提交，显示连接失败和诊断提示 |
| Redis 不可用 | 允许管理员选择暂缓，但标记部分功能不可用 |
| 文件目录不可写 | 阻止启用文件功能 |
| 初始化中断 | 未提交事务的数据回滚，可重新开始 |
| 已初始化后访问入口 | 返回登录页或 404，不展示初始化表单 |

### 4.1.8 验收标准

- 初始化成功后数据库存在组织、所有者和默认角色；
- 初始化接口不能被重复调用；
- 本地独立模式不会主动访问外部同步服务；
- 初始化失败不会产生半完成组织数据。

## 4.2 节点连接

### 4.2.1 功能描述

客户端通过地址、二维码或短连接码连接组织的 LanChat 节点。

### 4.2.2 页面入口

首次使用客户端时进入“连接节点”页面；用户也可以在设置中切换已保存节点。

### 4.2.3 基本流程

1. 用户选择手动输入、扫描二维码或选择历史节点；
2. 客户端请求节点公开信息接口；
3. 节点返回名称、组织名称、版本、证书摘要和可用能力；
4. 客户端展示节点确认信息；
5. 用户确认后保存节点；
6. 跳转对应登录页。

### 4.2.4 节点公开信息限制

未登录状态只允许返回：

- 节点名称；
- 组织名称；
- 节点版本；
- 登录方式；
- 证书摘要；
- 是否允许访客；
- 服务状态。

不得返回用户数量、内部地址、数据库信息和其他敏感配置。

### 4.2.5 节点状态

| 状态 | 展示 |
|---|---|
| 可连接 | 正常进入登录 |
| 连接超时 | 提示检查网络和地址 |
| 证书不可信 | 明确警告，不自动忽略 |
| 版本不兼容 | 提示升级客户端 |
| 节点维护中 | 展示维护说明 |
| 节点已停用 | 禁止登录 |

### 4.2.6 MVP 范围

MVP 支持：

- 手动地址；
- 二维码；
- 历史节点。

mDNS 自动发现和局域网广播发现放入桌面端阶段。

## 4.3 节点状态与运行模式

### 4.3.1 节点管理员视图

展示：

- 启动时间；
- 应用版本；
- 当前运行模式；
- 公网连接状态；
- 局域网地址；
- WebSocket 在线连接数；
- 在线用户数；
- 消息处理速率；
- 待同步任务；
- 数据库状态；
- Redis 状态；
- 文件存储状态；
- 磁盘剩余空间；
- 最近告警。

### 4.3.2 运行模式切换

组织管理员修改运行模式时：

1. 展示当前模式和目标模式；
2. 说明数据是否可能离开当前节点；
3. 展示将启用或停止的服务；
4. 进行权限二次确认；
5. 保存配置；
6. 写入审计日志；
7. 对正在执行的同步任务按规则暂停或继续。

禁止在未配置远程节点时启用自动同步。

## 5. 身份认证与设备管理

## 5.1 账号创建

### 5.1.1 创建方式

- 管理员单个创建；
- CSV 批量导入；
- 组织邀请码注册；
- 临时访客码。

### 5.1.2 管理员创建流程

1. 管理员填写用户名、昵称、角色和可选部门；
2. 设置初始密码或生成一次性密码；
3. 设置是否要求首次登录修改密码；
4. 服务端校验用户名唯一；
5. 创建用户；
6. 记录管理员操作日志；
7. 可复制一次性登录信息。

### 5.1.3 邀请码注册

- 邀请码绑定组织；
- 可绑定默认角色；
- 可设置总使用次数；
- 可设置有效期；
- 可限制允许加入的房间；
- 使用后必须记录设备、时间和结果；
- 失效邀请码不可再次使用。

### 5.1.4 异常处理

- 用户名重复；
- 邀请码过期；
- 邀请码次数耗尽；
- 组织已禁止自助注册；
- 访客码范围不匹配。

---

## 5.2 登录认证

### 5.2.1 登录方式

MVP 支持账号密码登录。短信、邮件和第三方 OAuth 不作为强制能力。

### 5.2.2 基本流程

1. 用户提交账号和密码；
2. 服务端按组织查找用户；
3. 校验账号状态；
4. 检查登录失败锁定；
5. 校验 BCrypt 密码；
6. 创建设备会话；
7. 签发 Access Token 和 Refresh Token；
8. 返回用户、组织、设备和权限摘要；
9. 客户端建立 WebSocket 连接；
10. WebSocket 完成认证后进入工作台。

### 5.2.3 登录状态

| 状态 | 行为 |
|---|---|
| 正常 | 允许登录 |
| 首次登录 | 强制修改密码 |
| 临时锁定 | 显示剩余锁定时间 |
| 管理员停用 | 禁止登录 |
| 访客过期 | 禁止登录 |
| 密码已过期 | 进入修改密码流程 |

### 5.2.4 Token 规则

- Access Token 短期有效；
- Refresh Token 与设备会话绑定；
- Refresh Token 轮换后旧令牌失效；
- 主动退出、设备下线或用户停用后对应会话失效；
- 前端不能把长期有效的 Refresh Token 暴露给普通脚本；
- WebSocket 不使用长期 Token 查询参数作为正式方案。

### 5.2.5 登录失败限制

- 默认连续 5 次错误后锁定 30 分钟；
- 计数按组织、账号和必要的设备/IP 维度控制；
- 成功登录后清除失败计数；
- 管理员可主动解锁；
- 返回信息不得泄露账号是否存在。

## 5.3 设备会话

### 5.3.1 设备列表

展示：

- 设备名称；
- 设备类型；
- 操作系统；
- 客户端版本；
- 首次登录时间；
- 最近活动时间；
- 最近连接节点；
- 在线状态；
- 可信状态；
- 当前设备标识。

### 5.3.2 主动下线

1. 用户点击下线设备；
2. 系统二次确认；
3. 服务端撤销设备会话；
4. 删除或失效 Refresh Token；
5. 向目标连接发送 `FORCE_LOGOUT`；
6. 关闭目标 WebSocket；
7. 目标端保留本地草稿，但禁止继续访问服务端；
8. 写入安全日志。

### 5.3.3 新设备提醒

用户在新设备登录后：

- 当前在线设备收到安全通知；
- 通知包含设备类型、时间和节点；
- 用户可立即下线该设备；
- 管理员可以配置是否要求已有可信设备确认。

## 5.4 访客账号

### 5.4.1 访客权限

访客只能访问被授权的房间，并可配置：

- 是否允许发送文本；
- 是否允许上传文件；
- 是否允许下载；
- 是否允许查看历史消息；
- 是否允许邀请其他成员；
- 到期时间。

### 5.4.2 到期处理

访客到期时：

1. 服务端撤销所有设备会话；
2. 关闭 WebSocket；
3. 从临时会话移除或标记失效；
4. 保留既有消息的发送者显示；
5. 禁止继续访问文件；
6. 记录到期处理日志。

## 6. 联系人、私聊与会话列表

## 6.1 组织成员搜索

### 6.1.1 搜索条件

支持：

- 用户 ID；
- 用户名；
- 昵称；
- 部门；
- 精确邀请码或房间成员选择。

### 6.1.2 隐私规则

普通用户只能看到：

- 头像；
- 昵称；
- 组织内显示名；
- 部门或岗位摘要；
- 在线状态是否可见由组织配置。

不得展示手机号、邮箱、IP 和设备信息。

### 6.1.3 限流

- 模糊搜索必须分页；
- 限制单用户搜索频率；
- 访客默认不能搜索全组织成员；
- 防止通过连续搜索枚举用户。

## 6.2 联系人管理

功能包括：

- 添加为常用联系人；
- 修改备注；
- 分组；
- 置顶；
- 免打扰；
- 拉黑；
- 取消拉黑。

组织可以配置两种私聊模式：

1. 组织成员可以直接私聊；
2. 必须先建立联系人或好友关系。

### 6.2.1 拉黑规则

- 被拉黑用户无法向拉黑者发送私聊；
- 已存在的聊天记录保留；
- 拉黑不影响双方在共同群组中的消息可见性；
- 是否允许查看对方基础资料由组织配置；
- 解除拉黑后不会自动恢复删除的联系人关系。

---

## 6.3 创建私聊

### 6.3.1 基本流程

1. 用户从成员列表选择目标用户；
2. 服务端校验双方属于同一组织；
3. 检查私聊策略和拉黑状态；
4. 查询是否已有 PRIVATE 会话；
5. 已有则返回原会话；
6. 无则创建会话和两条成员记录；
7. 返回会话详情；
8. 前端进入聊天页。

### 6.3.2 幂等规则

同一组织内，同一对用户只能存在一个有效私聊会话。创建操作必须具备数据库唯一约束或等价并发控制。

---

## 6.4 会话列表

### 6.4.1 排序规则

默认顺序：

1. 置顶会话；
2. 未处理紧急广播；
3. 按最后活动时间倒序。

### 6.4.2 会话条目内容

- 会话头像；
- 会话名称；
- 最后一条消息摘要；
- 最后活动时间；
- 未读数量；
- 待发送数量；
- 草稿摘要；
- 置顶标识；
- 免打扰标识；
- 同步失败标识；
- 房间即将到期标识。

### 6.4.3 未读数计算

推荐使用成员的 `last_read_sequence` 与会话 `last_sequence` 计算或维护聚合值。撤回、系统消息和不计未读的事件应按类型处理。

## 7. 群组与临时房间

## 7.1 普通群组

### 7.1.1 创建群组

创建字段：

- 群名称；
- 群头像；
- 群简介；
- 成员；
- 是否允许成员邀请；
- 加入方式；
- 历史消息可见范围；
- 默认文件权限。

创建者自动成为群主。

### 7.1.2 群角色

| 角色 | 权限 |
|---|---|
| 群主 | 全部群管理权限、转让和解散 |
| 管理员 | 成员管理、禁言、公告、广播 |
| 普通成员 | 收发消息和文件 |
| 只读成员 | 查看消息，不允许发送 |

### 7.1.3 添加成员

支持：

- 从组织成员中选择；
- 邀请链接；
- 邀请码；
- 二维码；
- 管理员审批。

添加前校验：

- 用户属于组织；
- 群未满员；
- 用户未被组织停用；
- 用户不在群内；
- 邀请人具有权限。

### 7.1.4 移除成员

- 群主可以移除管理员和普通成员；
- 管理员只能移除普通成员；
- 不可移除群主；
- 被移除用户立即失去发送、文件和历史补拉权限；
- 被移除用户收到系统通知；
- 离线发件箱中发往该群的消息恢复后应发送失败。

### 7.1.5 禁言

支持：

- 指定时长；
- 永久禁言；
- 解除禁言；
- 全体禁言；
- 仅管理员可发言。

禁言信息至少包含：

- 操作者；
- 原因；
- 开始时间；
- 结束时间；
- 当前状态。

### 7.1.6 群公告

- 群主和管理员可编辑；
- 发布后产生系统消息；
- 可要求成员确认；
- 支持查看历史版本；
- 修改公告需要记录审计。

### 7.1.7 退出与解散

- 普通成员可退出；
- 群主必须先转让群主；
- 解散后群状态变为已解散；
- 解散后禁止发送和补拉新数据；
- 历史数据根据组织保留策略归档或删除。

---

## 7.2 临时协作房间

### 7.2.1 创建字段

- 房间名称；
- 使用目的；
- 有效期；
- 成员上限；
- 加入方式；
- 是否允许访客；
- 是否允许成员邀请；
- 历史消息可见范围；
- 文件上传和下载策略；
- 是否允许转发；
- 到期处理方式；
- 是否允许外部同步。

### 7.2.2 加入方式

- 房间码；
- 二维码；
- 邀请链接；
- 管理员直接添加；
- 审核加入。

### 7.2.3 生命周期状态机

```text
DRAFT
  → ACTIVE
  → EXPIRING
  → FROZEN
  → ARCHIVED
  → DESTROY_PENDING
  → DESTROYED
```

说明：

- `DRAFT`：尚未开放加入；
- `ACTIVE`：正常协作；
- `EXPIRING`：即将到期；
- `FROZEN`：只读，不允许新消息和新文件；
- `ARCHIVED`：长期保留，只读访问；
- `DESTROY_PENDING`：等待执行销毁；
- `DESTROYED`：普通接口不可恢复。

### 7.2.4 到期流程

1. 到期前按配置提醒房间所有者；
2. 到期时停止新成员加入；
3. 根据策略冻结；
4. 执行归档或销毁任务；
5. 处理消息、附件和访问授权；
6. 更新搜索索引和缓存；
7. 形成审计记录；
8. 向相关成员发送结果通知。

### 7.2.5 销毁规则

- 删除消息正文；
- 删除无其他引用的物理文件；
- 撤销全部临时文件授权；
- 保留最低限度的审计记录；
- 销毁失败进入重试队列；
- 不得只修改状态而不处理实际数据。

## 8. 可靠消息系统

## 8.1 消息数据结构

每条消息至少包含：

| 字段 | 说明 |
|---|---|
| messageId | 服务端消息 ID |
| clientMsgId | 客户端生成的唯一 ID |
| organizationId | 组织 ID |
| conversationId | 会话 ID |
| sequence | 会话内递增序列号 |
| senderId | 发送者 ID |
| senderDeviceId | 发送设备 ID |
| messageType | 消息类型 |
| content | 消息内容或结构化载荷 |
| replyMessageId | 被引用消息 |
| status | 消息状态 |
| createdAt | 服务端创建时间 |
| clientCreatedAt | 客户端创建时间 |
| recalledAt | 撤回时间 |
| expireAt | 到期或焚毁时间 |

### 8.1.1 消息类型

- TEXT；
- IMAGE；
- FILE；
- SYSTEM；
- NOTICE；
- BROADCAST；
- RECALL_PLACEHOLDER；
- BURN_PLACEHOLDER；
- 以后可扩展 VOICE、VIDEO、LOCATION。

---

## 8.2 文本消息发送

### 8.2.1 前置条件

- 用户已登录；
- WebSocket 已认证，或使用允许的 HTTP 降级接口；
- 用户属于会话；
- 会话状态允许发送；
- 用户未被禁言；
- 文本长度符合限制。

### 8.2.2 客户端流程

1. 生成 `clientMsgId`；
2. 创建本地消息；
3. 状态设为 `SENDING` 或 `WAITING_NETWORK`；
4. 将消息写入本地持久化队列；
5. 通过 WebSocket 发送 `CHAT_SEND`；
6. 等待 `CHAT_ACK`；
7. 收到 ACK 后写入 `messageId` 和 `sequence`；
8. 状态更新为 `SENT`；
9. 超时未收到 ACK 时进入可重试状态。

### 8.2.3 服务端流程

1. 解析消息信封；
2. 校验协议版本；
3. 获取已认证用户和设备；
4. 校验组织和会话；
5. 校验成员、禁言和会话状态；
6. 根据用户和 `clientMsgId` 查询是否已处理；
7. 若已处理，返回原 ACK；
8. 若未处理，分配会话序列号；
9. 持久化消息；
10. 更新会话最后消息和最后序列号；
11. 事务提交；
12. 返回 ACK；
13. 发布推送事件；
14. 向在线接收设备投递。

### 8.2.4 业务规则

- 不使用客户端提交的 `senderId` 作为可信身份；
- 同一用户的 `clientMsgId` 必须唯一；
- 会话序列号由服务端生成；
- 服务端持久化失败时不得返回成功 ACK；
- 推送失败不应回滚已经成功持久化的消息；
- 文本内容必须防止 XSS；
- 不允许发送空白消息。

---

## 8.3 消息状态机

推荐状态：

```text
DRAFT
  → WAITING_NETWORK
  → SENDING
  → SENT
  → DELIVERED
  → READ

SENDING
  → FAILED
FAILED
  → SENDING

SENT / DELIVERED / READ
  → RECALLED

SENT / DELIVERED / READ
  → BURNED
```

### 8.3.1 状态说明

| 状态 | 含义 |
|---|---|
| DRAFT | 尚未提交 |
| WAITING_NETWORK | 客户端离线，等待连接 |
| SENDING | 已提交，等待服务端 ACK |
| SENT | 服务端已持久化 |
| DELIVERED | 至少目标设备确认收到 |
| READ | 接收用户已读 |
| FAILED | 发送失败，需要用户处理或重试 |
| RECALLED | 已撤回 |
| BURNED | 已焚毁 |

群聊的 `DELIVERED` 和 `READ` 可以使用聚合统计，不要求为每个成员实时展示完整明细。

---

## 8.4 ACK 与幂等

### 8.4.1 ACK 内容

```json
{
  "event": "CHAT_ACK",
  "requestId": "req_xxx",
  "payload": {
    "clientMsgId": "client_msg_xxx",
    "messageId": "server_msg_xxx",
    "conversationId": "conv_xxx",
    "sequence": 128,
    "serverTime": 1784077200108,
    "duplicated": false
  }
}
```

### 8.4.2 ACK 超时

- 客户端等待时间由配置决定；
- 超时后不能直接认为服务端未保存；
- 重试必须继续使用原 `clientMsgId`；
- 服务端发现重复请求时返回原处理结果；
- 客户端收到迟到 ACK 时合并状态，不创建第二条消息。

### 8.4.3 数据库约束

建议至少建立：

```sql
UNIQUE KEY uk_sender_client_msg (sender_id, client_msg_id);
UNIQUE KEY uk_conversation_sequence (conversation_id, sequence);
```

## 8.5 已送达和已读

### 8.5.1 私聊送达

接收设备成功写入本地消息缓存后发送送达确认。服务端聚合后通知发送者。

### 8.5.2 已读规则

- 用户打开会话并使消息进入已读范围后更新 `lastReadSequence`；
- 已读序列号只能前进，不能回退；
- 服务端向同一用户其他设备同步已读状态；
- 服务端向发送者推送已读更新；
- 免打扰不影响已读逻辑。

### 8.5.3 群聊已读

MVP 可仅记录每个成员的 `lastReadSequence`。展示时可显示“已读人数”，不要求为所有消息存储每人一条回执。

---

## 8.6 消息引用

- 引用消息保存原消息 ID；
- 展示发送者和摘要；
- 原消息撤回后显示“原消息已撤回”；
- 原文件权限失效后不得通过引用绕过权限；
- 跨会话引用默认禁止；
- 阅后即焚消息默认禁止引用。

---

## 8.7 消息撤回

### 8.7.1 基本流程

1. 用户选择撤回；
2. 客户端发送撤回请求；
3. 服务端校验消息发送者；
4. 校验会话关系；
5. 校验撤回时限；
6. 校验消息未被焚毁；
7. 更新消息为已撤回；
8. 创建撤回记录；
9. 撤销附件访问关系；
10. 事务提交后推送 `CHAT_RECALL`；
11. 各客户端替换为撤回占位。

### 8.7.2 文件处理

- 撤回文件消息时先撤销当前消息的附件引用；
- 文件仍被其他消息引用时不得删除物理文件；
- 引用数为零后进入异步清理；
- 已下载到本地的文件无法远程删除；
- 前端不得继续展示可用下载链接。

### 8.7.3 异常

- 超过时限；
- 非发送者；
- 消息不存在；
- 已撤回；
- 已焚毁；
- 用户已不在会话且无权操作。

---

## 8.8 阅后即焚

### 8.8.1 发送规则

支持文本、图片和短视频；文件和语音默认不支持。

发送前必须明确显示：

- 当前处于焚毁模式；
- 查看后倒计时时长；
- Web 端无法阻止截屏。

### 8.8.2 查看流程

1. 接收端请求查看内容；
2. 服务端校验会话和接收者；
3. 记录首次查看时间；
4. 返回内容和销毁时间；
5. 前端开始倒计时；
6. 倒计时结束或用户离开页面；
7. 客户端清除正文；
8. 服务端更新焚毁状态；
9. 向其他设备同步占位状态；
10. 按策略删除服务端正文。

### 8.8.3 多端规则

- 同一用户任意设备开始查看，可触发该用户维度的焚毁；
- 其他设备收到焚毁事件后清除本地内容；
- 离线设备上线后只能获得占位状态；
- 发送者端也不得永久保留已焚毁正文。

## 9. 断网可用与恢复

## 9.1 连接状态检测

客户端状态：

| 状态 | 含义 |
|---|---|
| CONNECTING | 正在建立连接 |
| AUTHENTICATING | 已连接，正在认证 |
| ONLINE | 已连接且同步完成 |
| DEGRADED | 部分服务不可用 |
| RECONNECTING | 正在重连 |
| OFFLINE | 无法连接节点 |
| SYNCING | 已连接，正在补拉数据 |

界面顶部应持续展示非正常状态。

---

## 9.2 本地缓存

### 9.2.1 缓存内容

- 最近会话；
- 最近消息；
- 草稿；
- 离线发件箱；
- 上传任务；
- 最后同步序列号；
- 节点信息；
- 用户和组织摘要。

### 9.2.2 存储要求

Web 端使用 IndexedDB 保存结构化数据。`localStorage` 仅保存非敏感的小型偏好配置。

### 9.2.3 数据清理

- 用户主动退出时根据安全策略清理；
- 设备被强制下线时删除敏感凭据；
- 访客账号到期后清除缓存；
- 超过缓存期限的消息可清理，但不能删除未发送消息；
- 清理前应保护草稿和失败任务。

---

## 9.3 离线发件箱

### 9.3.1 入队条件

- WebSocket 未连接；
- WebSocket 未认证；
- 节点不可达；
- 请求发送超时且未确认；
- 用户主动选择稍后发送。

### 9.3.2 队列条目

至少保存：

- `clientMsgId`；
- 会话 ID；
- 消息类型；
- 内容；
- 本地创建时间；
- 重试次数；
- 最近失败原因；
- 依赖的文件上传任务；
- 当前状态。

### 9.3.3 用户操作

- 查看待发送数量；
- 取消；
- 编辑文本；
- 立即重试；
- 批量重试；
- 删除失败项；
- 查看失败原因。

已经进入 `SENDING` 且服务端结果不确定的消息，不允许通过编辑生成同一 `clientMsgId` 的不同内容。

---

## 9.4 自动重连

### 9.4.1 策略

- 首次断开后快速重试；
- 后续使用指数退避；
- 增加随机抖动；
- 达到上限后降低频率；
- 浏览器重新获得网络或页面恢复时立即尝试；
- 用户可手动重连。

### 9.4.2 重连流程

```text
检测网络
→ 建立 WebSocket
→ 认证设备会话
→ 请求全局状态
→ 按会话补拉遗漏消息
→ 合并已读、撤回、焚毁和权限状态
→ 处理离线发件箱
→ 恢复文件任务
→ 标记 ONLINE
```

客户端不能在遗漏消息尚未合并时直接把状态显示为“同步完成”。

---

## 9.5 遗漏消息补拉

### 9.5.1 请求数据

客户端提交每个活跃会话的最后确认序列号。

### 9.5.2 服务端响应

- 返回缺失消息；
- 返回最新会话序列号；
- 返回撤回和焚毁状态；
- 返回成员权限变化；
- 数据量过大时分页；
- 超出保留期时返回“历史已过期”标识。

### 9.5.3 客户端合并

按以下优先级合并：

1. 权限撤销；
2. 会话解散、冻结和销毁；
3. 消息撤回和焚毁；
4. 服务端正式消息；
5. 本地待发送消息；
6. 草稿和临时 UI 状态。

---

## 9.6 冲突处理

### 9.6.1 权限冲突

用户离线期间被移出群组：

- 本地仍可查看缓存；
- 不允许补拉新消息；
- 待发送消息失败；
- 文件授权失效；
- 前端显示“已不在该群组”。

### 9.6.2 草稿冲突

MVP 采用“当前设备草稿优先，本地保留其他副本”的策略，不进行复杂实时协同编辑。

### 9.6.3 设置冲突

置顶、免打扰等使用服务端更新时间或版本号解决。冲突时保留最新成功提交值。

### 9.6.4 节点同步冲突

自动无法解决的冲突进入管理员列表，展示：

- 数据类型；
- 本地版本；
- 远程版本；
- 修改时间；
- 操作者；
- 建议处理方式。

## 10.2 SHA-256 秒传

### 10.2.1 流程

1. 客户端计算 SHA-256；
2. 调用文件检查接口；
3. 服务端查询相同哈希的物理文件；
4. 未找到则返回需要上传；
5. 找到则检查文件状态和安全扫描结果；
6. 创建当前用户和会话的文件引用；
7. 返回新文件授权信息；
8. 创建文件消息。

### 10.2.2 安全规则

相同哈希只说明物理内容相同，不代表用户拥有访问权限。秒传接口不得返回其他用户上传记录、文件路径或原始授权。

## 10.3 分片上传

### 10.3.1 上传状态机

```text
CREATED
  → UPLOADING
  → PAUSED
  → UPLOADING
  → MERGING
  → VERIFYING
  → COMPLETED

UPLOADING / MERGING / VERIFYING
  → FAILED

CREATED / PAUSED / FAILED
  → CANCELED
```

### 10.3.2 创建任务

返回：

- uploadId；
- 分片大小；
- 分片总数；
- 并发限制；
- 已存在分片；
- 过期时间。

### 10.3.3 上传分片

- 每个分片带序号和校验信息；
- 支持重复提交同一分片；
- 重复分片不得重复计数；
- 限制单用户并发；
- 分片完成后更新任务进度。

### 10.3.4 合并和校验

1. 检查全部分片；
2. 获取合并锁；
3. 顺序合并；
4. 计算最终 SHA-256；
5. 与客户端哈希比较；
6. 写入对象存储；
7. 创建文件对象；
8. 删除临时分片；
9. 返回完成结果。

### 10.3.5 异常处理

- 分片缺失；
- 分片校验失败；
- 任务过期；
- 存储空间不足；
- 合并中断；
- 最终哈希不一致；
- 文件被安全扫描隔离。

### 10.4.3 用户展示

## 17. WebSocket 事件分析

## 17.1 通用信封

```json
{
  "version": 1,
  "event": "CHAT_SEND",
  "requestId": "req_xxx",
  "timestamp": 1784077200000,
  "payload": {}
}
```

## 17.2 核心事件表

| 事件 | 方向 | 说明 |
|---|---|---|
| AUTH | 客户端 → 服务端 | WebSocket 认证 |
| AUTH_OK | 服务端 → 客户端 | 认证成功 |
| PING | 双向 | 心跳探测 |
| PONG | 双向 | 心跳响应 |
| CHAT_SEND | 客户端 → 服务端 | 发送消息 |
| CHAT_ACK | 服务端 → 客户端 | 持久化确认 |
| CHAT_DELIVER | 客户端 → 服务端 | 消息送达确认 |
| CHAT_READ | 客户端 → 服务端 | 会话已读序列 |
| CHAT_RECALL | 双向 | 撤回请求或推送 |
| CHAT_BURN | 双向 | 焚毁请求或推送 |
| TYPING_START | 双向 | 正在输入 |
| TYPING_STOP | 双向 | 停止输入 |
| SYNC_REQUEST | 客户端 → 服务端 | 请求遗漏数据 |
| SYNC_RESPONSE | 服务端 → 客户端 | 返回同步数据 |
| BROADCAST | 服务端 → 客户端 | 广播推送 |
| BROADCAST_ACK | 客户端 → 服务端 | 广播确认 |
| PERMISSION_CHANGED | 服务端 → 客户端 | 权限变化 |
| ROOM_STATUS_CHANGED | 服务端 → 客户端 | 房间状态变化 |
| FORCE_LOGOUT | 服务端 → 客户端 | 强制下线 |
| TOKEN_EXPIRED | 服务端 → 客户端 | 认证过期 |
| ERROR | 服务端 → 客户端 | 协议或业务错误 |

## 17.3 事件处理要求

- 每个请求型事件携带 `requestId`；
- 消息发送携带 `clientMsgId`；
- 客户端忽略未知但非致命事件；
- 协议版本不兼容时明确提示升级；
- 单连接消息速率超过限制时进行限流；
- 无法解析的事件不得导致 WebSocket 服务线程崩溃；
- 错误响应不得包含堆栈和数据库信息。

| outbox_event | 事务后事件 |

## 18.2 关键索引

```sql
UNIQUE(sender_id, client_msg_id)
UNIQUE(conversation_id, sequence)
INDEX(conversation_id, sequence DESC)
INDEX(user_id, last_activity_at)
INDEX(organization_id, status)
INDEX(status, expire_at)
INDEX(file_hash)
INDEX(upload_id, part_number)
INDEX(broadcast_id, confirm_status)
INDEX(event_type, created_at)
```

## 18.3 数据一致性

以下操作需要事务保护：

- 创建私聊会话和会话成员；
- 发送消息和更新会话最后序列；
- 撤回消息和撤销附件引用；
- 创建广播和接收者记录；
- 创建文件引用和文件消息；
- 房间到期状态变更；
- 设备会话撤销。

消息推送建议在事务提交后执行，避免客户端收到尚未提交的数据。

## 19. Redis 与异步任务功能

## 19.1 Redis 用途

- 登录失败计数；
- 短期验证码或邀请码状态；
- 设备会话快速校验；
- 在线用户和连接路由；
- WebSocket 实例映射；
- 接口和消息限流；
- 临时预览 Token；
- 短期幂等缓存；
- 分布式锁；
- 热点会话摘要。

永久消息和关键权限不能只保存在 Redis。

## 19.2 异步任务

适合异步执行：

- WebSocket 推送；
- 文件合并；
- 缩略图生成；
- 文件安全扫描；
- Office 转换；
- 离线通知；
- 文件清理；
- 临时房间销毁；
- 节点同步；
- 审计归档；
- 统计聚合。

异步任务需要：

- 任务状态；
- 重试次数；
- 最大重试；
- 失败原因；
- 死信或人工处理；
- 幂等键。

## 21. 异常场景汇总

| 场景 | 处理方式 |
|---|---|
| 节点地址错误 | 提示无法连接并保留输入 |
| 证书不可信 | 阻止静默连接，显示风险 |
| 登录失败 | 返回统一错误，不泄露账号存在性 |
| WebSocket 认证失败 | 关闭连接并跳转登录或刷新会话 |
| 发送消息时断网 | 进入离线发件箱 |
| ACK 丢失 | 使用同一 `clientMsgId` 重试 |
| 重复消息请求 | 返回原消息 ACK |
| 用户已被移出群组 | 拒绝发送并更新本地权限状态 |
| 会话已冻结 | 只允许查看，不允许发送 |
| 临时房间到期 | 按策略冻结、归档或销毁 |
| 撤回超时 | 拒绝并显示超时 |
| 阅后即焚倒计时中客户端关闭 | 再次进入时以服务端状态为准 |
| 文件哈希存在但无权限 | 只复用物理文件，不授予其他资源权限 |
| 分片上传断网 | 保留分片，恢复后继续 |
| 文件直传失败 | 降级节点中转 |
| 存储空间不足 | 停止新文件任务，文本聊天继续 |
| Redis 不可用 | 在线状态、限流等降级并告警 |
| 预览服务不可用 | 有权限时提供下载 |
| 多设备已读不一致 | 以最大已读序列同步 |
| 同步冲突 | 自动规则处理或进入管理员冲突列表 |
| 设备被下线 | 清理凭据、关闭连接、保留草稿 |
| 广播重复确认 | 幂等更新，不重复计数 |
| 浏览器休眠后恢复 | 重新心跳、重连并补拉遗漏消息 |
| SPA 子路由刷新 | 后端回退至前端 `index.html`，排除 `/api` 和 `/ws` |

## 25. V2.0 修订说明

本次重构相较 V1.0 的主要变化：

1. 产品中心由普通网络聊天室调整为 LAN-first 私有即时协作系统；
2. 新增组织、节点和运行模式；
3. 将私聊、群聊和临时房间统一为会话模型；
4. 增加可靠消息协议、ACK、幂等和会话序列号；
5. 增加断网发件箱、自动重连和遗漏消息补拉；
6. 增加局域网传输路径和失败降级；
7. 增加临时协作房间和完整生命周期；
8. 增加应急广播及确认统计；
9. 增加可选节点同步和冲突处理；
10. 增加网络诊断、管理后台和可观测能力；
11. 增加文件对象与消息引用分离的权限模型；
12. 增加前端、WebSocket、数据库、Redis 和异步任务的实现要求；
13. 增加 MVP 验收清单和分阶段实施顺序；
14. 保留消息撤回、阅后即焚、已读、免打扰等原有能力，但重新纳入可靠消息和私有协作体系。
