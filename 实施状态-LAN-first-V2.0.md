# LanChat V2.0 实施状态

更新日期：2026-07-15

本文把《需求分析-LAN-first-V2.0》和《功能分析-LAN-first-V2.0》的目标能力映射到当前代码。状态只描述已有证据，不把设计稿或性能目标算作完成结果。

## 1. 本轮交付范围

本轮完成的是 P0 的“可靠单节点聊天”纵向切片：

1. 统一私聊与群聊会话标识；
2. `clientMsgId` 幂等、会话序列号和事务提交后 ACK；
3. 按序列游标查询历史与重连增量补拉；
4. IndexedDB 本地消息缓存与文本离线发件箱；
5. WebSocket 连接后认证、心跳、指数退避和设备会话吊销；
6. Refresh Token 轮换与 HttpOnly Cookie；
7. 文件会话权限、物理对象授权和短期签名预览；
8. 单节点 Docker/Compose 部署骨架；
9. 连接诊断条与关键自动化测试。

## 3. 数据模型变更

### 新增

- `conversation`：统一私聊、群聊以及未来临时房间/广播的顶层模型；
- `conversation_member`：角色、最后已读序列、未读数和离开时间；
- `file_access_grant`：把物理文件存在与用户引用权限分开。

### `chat_message` 新字段

- `client_msg_id`
- `conversation_id`
- `sequence`
- `sender_device_id`
- `client_created_at`

已有数据库必须执行 `sql/migration-v2.0-reliable-messaging.sql`。迁移前需备份；新环境直接使用 `sql/init.sql`。

## 4. 关键工程取舍

### 单节点优先

当前保持模块化单体，不提前拆微服务。WebSocket Session 仍在单个 JVM 内；跨实例路由必须在引入 Redis Streams、RabbitMQ 或 Outbox 后再声明支持。

### 文本先离线

文本消息可以在无网络时进入 IndexedDB 发件箱。文件仍需先连接节点上传，因此 UI 会在离线时禁用文件操作，避免展示虚假的“已离线保存上传任务”。

会话目录和最近消息也保存在 IndexedDB，因此已认证标签页断网刷新后仍可进入本地会话。完全关闭浏览器会清除 `sessionStorage` 中的短期 Access Token；当前版本没有本地 PIN/设备密钥解锁机制，此时需要重新连到节点完成认证后才能进入缓存，不能宣称“冷启动完全离线登录”。

### ACK 的含义

`CHAT_ACK` 表示服务端已经完成权限校验和数据库事务提交，不代表对方已经阅读。发送方重试相同 `clientMsgId` 时返回原消息信息，不重复落库。

### 文件秒传权限

`/file/check` 只有在用户已经拥有该物理文件的访问权时才返回秒传结果。用户上传完整文件并由服务端重新计算 SHA-256 后，可以获得 `UPLOAD_PROOF` 授权。仅知道哈希或存储名不能引用文件。网页与可执行扩展名默认拒绝，非受控媒体预览会强制以二进制附件下载，避免同源脚本执行。

## 5. 当前可验证验收项

- 相同发送者重复提交同一 `clientMsgId` 不会生成第二条消息；
- 每个会话的消息以唯一递增 `sequence` 排序；
- WebSocket URL 不含访问令牌，10 秒内未发 `AUTH` 会断开；
- 设备被下线后，后续事件或心跳会收到强制退出；
- 浏览器断网时可继续发送文本并看到待发数量；
- 恢复连接后先补拉缺失消息，再按创建时间补发本地队列；
- 单次同步不足以返回全部消息时会继续按新位置分页；
- Refresh Token 不出现在 JSON 和 Web Storage；
- 普通令牌过期会保留属主隔离的离线发件箱，重新认证同一账号后继续；明确退出或设备强制下线会清理本地数据；
- 无会话权限不能上传附件，无文件权限不能发送附件消息；
- 在线列表不会返回密码和设备令牌。

## 6. 下一阶段建议

按价值和依赖关系推进：

1. Testcontainers + 两个 WebSocket 客户端的真实 MySQL/Redis 集成测试；
2. 分片上传、上传会话、文件引用表与 MinIO；
3. 局域网节点连接/发现与 WebRTC DataChannel 直传原型；
4. 临时协作房间和应急广播；
5. Outbox + Redis Streams/RabbitMQ 的跨实例路由；
6. Actuator、Prometheus、Grafana 和实际压测数据。

在这些能力完成并经过测试前，不应在简历或 README 中声称“5000 条/秒”“99.9% 可用”“支持多节点”“WebRTC 直传”或“生产级”。
